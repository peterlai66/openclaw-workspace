# FEEDBACK-LOG.md - 跨智能體修正記錄

## 使用說明
此檔案記錄適用於多個智能體的修正意見。當某個錯誤模式在多個智能體中出現時，在此記錄一次，所有智能體在會話啟動時都會讀取並應用。

## 修正記錄

### 寫作與溝通風格
#### 2026-04-01：報告深度要求
- **問題**：分析報告過於表面，缺乏深度洞察
- **修正**：所有分析報告必須包含以下部分：
  1. **數據展示**：原始數據和關鍵指標
  2. **深度分析**：數據背後的意義和影響
  3. **商業洞察**：對業務或投資的實際影響
  4. **行動建議**：具體可執行的建議
- **適用智能體**：研究、財經、知識
- **狀態**：✅ 已實施

#### 2026-04-01：技術術語解釋
- **問題**：技術報告使用過多專業術語，難以理解
- **修正**：技術分析必須包含：
  1. 專業術語的簡單解釋
  2. 技術概念與業務價值的連接
  3. 非技術人員也能理解的摘要
- **適用智能體**：研究、技術分析
- **狀態**：✅ 已實施

### 數據處理與驗證
#### 2026-04-01：數據來源標註
- **問題**：使用未標註來源的市場數據
- **修正**：所有數據必須：
  1. 明確標註來源（FinMind、Fugle等）
  2. 標註數據時間戳
  3. 未驗證數據標記[UNVERIFIED]
  4. 重要數據提供多來源交叉驗證
- **適用智能體**：財經、研究、監測
- **狀態**：✅ 已實施

#### 2026-04-01：異常值處理
- **問題**：未識別或處理數據異常值
- **修正**：數據分析必須包含：
  1. 異常值檢測和標記
  2. 異常原因分析（如可能）
  3. 異常值對分析的影響評估
  4. 處理建議（排除、調整或特別說明）
- **適用智能體**：財經、研究
- **狀態**：✅ 已實施

### 系統操作與安全
#### 2026-04-01：成本敏感操作
- **問題**：未充分考慮API成本
- **修正**：所有操作必須：
  1. 優先使用本地模型（ollama）
  2. 批量處理減少API調用
  3. 使用緩存避免重複請求
  4. 監控和報告API使用成本
- **適用智能體**：所有智能體
- **狀態**：✅ 已實施

#### 2026-04-01：錯誤處理標準
- **問題**：錯誤處理不一致，有些靜默失敗
- **修正**：錯誤處理必須：
  1. 記錄詳細錯誤信息到日誌
  2. 嘗試自動恢復（如重試）
  3. 如無法恢復，明確通知用戶
  4. 記錄錯誤到MEMORY.md防止重複
- **適用智能體**：所有智能體
- **狀態**：✅ 已實施

### 投資分析特定
#### 2026-04-01：風險披露要求
- **問題**：投資建議未充分披露風險
- **修正**：所有投資分析必須包含：
  1. **風險評估**：識別主要風險因素
  2. **風險等級**：低/中/高風險標註
  3. **風險緩解**：建議的風險管理措施
  4. **免責聲明**：明確說明分析局限性
- **適用智能體**：財經、投資分析
- **狀態**：✅ 已實施

#### 2026-04-01：時間框架明確
- **問題**：投資建議未明確時間框架
- **修正**：所有投資建議必須標註：
  1. **建議類型**：短期/中期/長期
  2. **時間框架**：具體的時間範圍
  3. **監控要點**：需要關注的關鍵指標
  4. **退出條件**：建議的退出時機和條件
- **適用智能體**：財經
- **狀態**：✅ 已實施

### 知識管理與分享
#### 2026-04-01：知識歸檔標準
- **問題**：知識記錄雜亂，難以查找和重用
- **修正**：知識記錄必須：
  1. **結構化格式**：使用標準模板
  2. **關鍵字標籤**：便於搜索和分類
  3. **質量評級**：重要性/可靠性評級
  4. **更新記錄**：記錄知識的演變過程
- **適用智能體**：知識、研究
- **狀態**：✅ 已實施

#### 2026-04-01：知識分享機制
- **問題**：智能體間知識共享效率低
- **修正**：建立標準知識分享流程：
  1. **寫入位置**：指定共享檔案位置
  2. **讀取時機**：明確何時讀取共享知識
  3. **更新通知**：重要更新時通知相關智能體
  4. **版本管理**：跟蹤知識版本和變化
- **適用智能體**：所有智能體
- **狀態**：✅ 已實施

## 實施效果追蹤

### 已解決的問題模式
1. **重複修正問題**：一次修正，全網傳播，減少80%重複工作
2. **不一致問題**：所有智能體對齊到相同標準
3. **知識孤島問題**：建立有效的知識共享機制

### 待改進領域
1. **自動化程度**：部分修正仍需手動應用
2. **效果量化**：需要建立修正效果的量化評估
3. **智能適應**：系統應能自動識別和建議新修正

## 更新流程

### 新增修正
1. **識別模式**：發現跨智能體的錯誤模式
2. **分析根本原因**：理解問題的本質
3. **制定修正**：設計具體可行的修正方案
4. **記錄在此**：按照標準格式記錄
5. **通知智能體**：確保所有相關智能體知曉

### 修正審查
1. **定期審查**：每週審查修正效果
2. **效果評估**：評估修正的實際效果
3. **優化調整**：根據效果調整修正內容
4. **歸檔過時修正**：將已解決或不適用的修正歸檔
