# 正確任務流程
## 防止回到舊的錯誤工作模式

**版本**: 1.0  
**創建時間**: 2026-04-04  
**最後更新**: 2026-04-04 21:51

---

## 🚨 **問題背景**
經常回到舊的錯誤工作模式：
1. 規劃而不是執行
2. 等待確認而不是行動
3. 分析而不是執行
4. 手動檢查而不是自動監控
5. 說「X分鐘後」但忘記

---

## 🔄 **正確任務流程圖**

```
收到任務 → 評估複雜度 → 是否超過2分鐘?
    ↓是                           ↓否
立即確認「收到,開Task處理中」 → 直接處理並回應
    ↓
啟動 sessions_spawn (子代理)
    ↓
繼續與用戶聊天互動
    ↓
子代理完成 → 整合結果 → 轉交用戶
```

---

## 📋 **強制執行規則**

### 規則1：所有超過2分鐘任務必須開Task
- **零例外政策**：任何預計超過2分鐘的任務
- **立即確認**：3秒內必須回應確認
- **智能路由**：根據複雜度選擇合適模型
- **背景處理**：不讓用戶等待

### 規則2：簡單任務直接處理
- **2分鐘內任務**：直接處理立即回應
- **即時查詢**：立即回答不拖延
- **輕量操作**：文件讀寫、簡單檢查

### 規則3：永不規劃立即執行
- **禁止詞彙**：今天、明天、本週、下週、下個月、規劃、計劃
- **立即行動**：聽到需求立即執行
- **事後回報**：行動完成後報告結果

### 規則4：建立自動監控不手動檢查
- **監控系統**：建立自動化監控
- **自動報告**：定期自動發送報告
- **問題檢測**：自動發現和警報
- **不手動檢查**：等待自動通知

## 🛠️ **具體實施方法**

### 1. 任務複雜度評估
```bash
# 使用智能路由系統
bash ~/.openclaw/workspace/scripts/model-router.sh "任務描述" "complexity" "channel"
```

### 2. 子代理啟動模板
```bash
sessions_spawn \
  --task "任務描述" \
  --label "有意義的標籤" \
  --runtime "subagent" \
  --model "合適的模型" \
  --timeoutSeconds 300
```

### 3. 互動體驗優化
- **立即確認**：0-3秒內回應
- **分段輸出**：長時間任務分段更新
- **不等待**：子代理執行時繼續互動
- **完成總結**：整合結果精煉轉交

### 4. 監控系統集成
- **每5分鐘**：自動監控報告
- **自動發送**：Discord自動報告
- **問題警報**：自動檢測和通知
- **持續優化**：基於數據改進

## 📊 **錯誤模式與修正**

### 錯誤模式1：規劃而不是執行
- **錯誤表現**：說「我們可以規劃...」、「建議明天...」
- **正確做法**：說「立即執行...」、「現在就做...」
- **觸發規則**：聽到「規劃」→ 觸發「不做行程規劃，立即執行」

### 錯誤模式2：等待確認而不是行動
- **錯誤表現**：說「等你確認」、「需要你同意」
- **正確做法**：說「主動行動，事後回報」
- **觸發規則**：聽到「確認」→ 觸發「不用等你確認，直接執行」

### 錯誤模式3：分析而不是執行
- **錯誤表現**：說「需要分析...」、「先研究一下」
- **正確做法**：說「執行中分析」、「現在開始做」
- **觸發規則**：聽到「分析」→ 觸發「不分析等待，立即執行」

### 錯誤模式4：手動檢查而不是自動監控
- **錯誤表現**：說「我檢查一下」、「手動查看」
- **正確做法**：說「建立自動監控」、「等待自動通知」
- **觸發規則**：聽到「檢查」→ 觸發「等待自動通知，不是手動檢查」

### 錯誤模式5：時間承諾忘記
- **錯誤表現**：說「5分鐘後」但忘記
- **正確做法**：立即記錄到時間承諾系統
- **觸發規則**：聽到「分鐘後」→ 觸發「立即記錄到時間承諾系統」

## 🧠 **記憶觸發集成**

### 觸發詞彙與對應行動
1. **「cron」** → 用task系統，不是cron
2. **「規劃」** → 不做行程規劃，立即執行
3. **「確認」** → 不用等你確認，直接執行
4. **「分析」** → 不分析等待，立即執行
5. **「檢查」** → 等待自動通知，不是手動檢查
6. **「分鐘後」** → 立即記錄到時間承諾系統

### 行為修正規則
1. 每次說「X分鐘後」→ 必須記錄到時間承諾系統
2. 每次創建任務→ 必須設定正確的通知策略
3. 每次承諾回報→ 必須建立實際的提醒機制
4. 每次違反原則→ 必須立即自我修正並記錄學習
