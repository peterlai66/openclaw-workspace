## 🚨 **問題背景**
多次犯錯說「X分鐘後」但忘記執行，導致用戶等待無果。

## 📋 **強制規則**
1. **每次說「X分鐘後」** → 必須立即記錄到此系統
2. **每次承諾時間** → 必須建立實際提醒機制
3. **每次完成承諾** → 必須標記完成狀態
4. **每次違反承諾** → 必須記錄錯誤並修正

## 📝 **承諾記錄流程**

### 1. 創建新承諾
```bash
# 當說「X分鐘後」時立即執行
echo "$(date '+%Y-%m-%d %H:%M') | X分鐘後做某事 | $(date -d 'X minutes' '+%H:%M') | 待完成 | 進行中 | 提醒機制" >> time-promises.md
```

### 2. 更新承諾狀態
```bash
# 完成時更新
sed -i "s/| 待完成 |/| $(date '+%H:%M') | ✅ 完成 |/" time-promises.md
```

### 3. 檢查過期承諾
```bash
# 每天檢查未完成承諾
grep "待完成" time-promises.md
```

## 🔔 **提醒機制實現**

### 自動提醒方法
1. **Cron任務** - 定時執行承諾
2. **監控系統** - 自動檢查和報告
3. **會話啟動** - 每次啟動檢查未完成承諾
4. **心跳檢查** - 定期檢查承諾狀態

### 具體實現
```bash
# 檢查並提醒未完成承諾
check_promises() {
    local current_time=$(date '+%H:%M')
    grep "待完成" time-promises.md | while read line; do
        local promise_time=$(echo "$line" | awk -F'|' '{print $3}' | tr -d ' ')
        if [[ "$current_time" > "$promise_time" ]]; then
            echo "🚨 過期承諾: $line"
            # 發送提醒
        fi
    done
}
```
