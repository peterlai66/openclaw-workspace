# 任務設計檢查清單

## 創建前檢查（必須全部通過）

### 1. 標識檢查
- [ ] **label清晰**：能一眼看出任務目的（如「系統優化：清理舊日誌」）
- [ ] **描述明確**：簡要描述任務要做什麼
- [ ] **唯一標識**：避免與現有任務重複

### 2. 資源檢查
- [ ] **超時合理**：根據複雜度設置合理超時（簡單:15分鐘, 中等:1小時, 複雜:4小時）
- [ ] **權限足夠**：確認有執行任務所需的權限
- [ ] **資源可用**：確認所需資源（CPU、記憶體、儲存）可用

### 3. 設計檢查
- [ ] **目標明確**：具體可衡量的成功標準
- [ ] **步驟清晰**：有明確的執行步驟
- [ ] **產出定義**：明確任務要產出什麼

## 執行中檢查（定期監控）

### 4. 進度監控
- [ ] **進度報告**：定期報告執行進度
- [ ] **時間控制**：在超時範圍內執行
- [ ] **資源監控**：監控資源使用情況

### 5. 問題處理
- [ ] **錯誤處理**：遇到錯誤有處理機制
- [ ] **恢復方案**：失敗後有恢復計劃
- [ ] **警報機制**：嚴重問題及時警報

## 完成後檢查（確保質量）

### 6. 成果驗證
- [ ] **目標達成**：檢查是否達到成功標準
- [ ] **產出完整**：檢查產出是否完整可用
- [ ] **質量檢查**：檢查產出質量

### 7. 清理和學習
- [ ] **資源清理**：正確清理使用的資源
- [ ] **學習記錄**：記錄學到的經驗教訓
- [ ] **優化建議**：提出改進建議

## 檢查工具使用：

### 創建任務前：
```bash
# 使用檢查清單
cat system/task-design-checklist.md | grep "創建前檢查" -A 20

# 確認所有項目都通過
```

### 執行任務時：
```bash
# 監控進度
openclaw tasks list --status running

# 檢查資源
top -bn1 | head -5
```

### 完成任務後：
```bash
# 驗證成果
ls -la 產出檔案

# 記錄學習
echo "學習記錄：$(date)" >> memory/學習記錄.md
```

## 常見問題和解決方案：

### 問題1：任務卡住
**原因**：無超時設置或超時不合理
**解決**：設置合理超時，添加進度檢查點

### 問題2：無產出
**原因**：目標不明確，無產出定義
**解決**：明確定義具體產出，定期檢查產出

### 問題3：資源耗盡
**原因**：資源評估不足
**解決**：創建前檢查資源，執行中監控資源
