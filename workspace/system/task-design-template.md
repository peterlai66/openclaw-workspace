# 任務設計模板

## 任務標題
**清晰描述任務目的**

## 任務設計檢查清單
### 創建前檢查：
- [ ] **label清晰**：能一眼看出任務目的
- [ ] **超時合理**：根據複雜度設置合理超時
- [ ] **目標明確**：具體可衡量的成功標準
- [ ] **資源評估**：需要的資源和權限
- [ ] **風險評估**：可能遇到的問題和對策

### 執行設計：
- [ ] **進度檢查點**：定期報告進度的設計
- [ ] **錯誤處理**：遇到錯誤的處理機制
- [ ] **恢復方案**：失敗後的恢復計劃
- [ ] **清理機制**：完成後的資源清理

### 完成後審查：
- [ ] **成果驗證**：檢查是否達到目標
- [ ] **學習記錄**：記錄學到的經驗
- [ ] **優化建議**：提出改進建議
- [ ] **知識歸檔**：歸檔到知識庫

## 模板使用示例：

```markdown
# 系統優化：清理舊日誌檔案

## 任務設計檢查清單
### 創建前檢查：
- [x] **label清晰**：系統優化：清理舊日誌檔案
- [x] **超時合理**：1小時（清理可能較慢）
- [x] **目標明確**：清理超過30天的日誌檔案
- [x] **資源評估**：需要檔案系統讀寫權限
- [x] **風險評估**：避免刪除重要日誌

### 執行設計：
- [x] **進度檢查點**：每10分鐘報告清理進度
- [x] **錯誤處理**：遇到權限問題跳過並記錄
- [x] **恢復方案**：保留刪除清單可恢復
- [x] **清理機制**：完成後釋放記憶體資源

### 具體行動：
1. **掃描日誌目錄**（0-10分鐘）
2. **篩選舊檔案**（10-20分鐘）
3. **安全刪除**（20-40分鐘）
4. **生成報告**（40-50分鐘）
5. **清理資源**（50-60分鐘）

### 進度報告：
- 開始時間：{timestamp}
- 當前階段：{current_phase}
- 已完成：{completed_percent}%
- 預計完成：{eta}

### 成功標準：
- ✅ 清理超過30天的日誌檔案
- ✅ 無重要日誌被誤刪
- ✅ 生成清理報告
- ✅ 資源正確清理

### 學習記錄：
- 學到的經驗
- 遇到的問題和解決方案
- 優化建議
```

## 良好設計 vs 不良設計：

### ❌ 不良設計示例：
```
任務：做一些優化
```
- 無label
- 無超時
- 無進度報告
- 無明確目標

### ✅ 良好設計示例：
```
任務：系統優化：清理舊日誌檔案（超時：1小時）
```
- 清晰label
- 合理超時
- 定期進度報告
- 明確成功標準

## 我的承諾：
**從現在開始，只創建良好設計的任務，從源頭避免卡住和錯誤問題。**
