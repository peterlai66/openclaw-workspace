# 工作驗證系統

## 原則：
**不問「夠清楚了嗎」，建立可自我驗證的工作成果**

## 驗證標準：

### 1. 工具類工作
**必須驗證**：
- ✅ 檔案存在且可執行
- ✅ 功能正常運作
- ✅ 有使用說明或示例
- ✅ 錯誤處理完善

**驗證方法**：
```bash
# 檢查檔案存在
ls -la /path/to/tool

# 測試功能
./tool --test 或 ./tool --help

# 檢查輸出
./tool | head -5
```

### 2. 系統類工作
**必須驗證**：
- ✅ 系統狀態改善
- ✅ 問題解決證明
- ✅ 性能提升數據
- ✅ 可持續運作

**驗證方法**：
```bash
# 檢查系統狀態
openclaw tasks audit

# 檢查資源使用
./scripts/trend-analysis.sh

# 檢查問題解決
grep -r "已解決" system/
```

### 3. 報告類工作
**必須驗證**：
- ✅ 數據準確性
- ✅ 內容完整性
- ✅ 可讀性良好
- ✅ 有實際價值

**驗證方法**：
```bash
# 檢查報告存在
find . -name "*報告*.md" -type f

# 檢查內容質量
head -20 報告檔案.md

# 檢查數據來源
grep -E "(數據|來源|驗證)" 報告檔案.md
```

## 我的工作流程：

### 收到任務時：
1. ✅ **分析需求** → 自己分析，不問確認
2. ✅ **制定方案** → 自己設計，不問意見
3. ✅ **執行實施** → 直接行動，不問許可
4. ✅ **驗證成果** → 自我驗證，不問評價
5. ✅ **回報結果** → 提供事實，不問確認

### 完成工作時：
1. ✅ **提供具體成果**：檔案、數據、報告
2. ✅ **提供驗證方法**：如何檢查工作成果
3. ✅ **提供改進建議**：下一步優化方向
4. ✅ **不問確認問題**：相信自己的工作質量
